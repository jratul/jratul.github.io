프로그램이 종료되면 메모리의 데이터는 사라집니다. 데이터를 영구적으로 보존하려면 **파일**에 저장해야 합니다. Java는 파일을 읽고 쓰는 다양한 도구를 제공합니다.

---

## Java I/O 방식의 변천사

```
java.io    (전통 방식)  → 스트림 기반, 블로킹, 복잡함
java.nio   (Java 4+)    → 버퍼, 채널 기반
java.nio.file (NIO.2, Java 7+) → Path, Files API — 현재 권장 방식
```

현재는 대부분 `java.nio.file` 패키지의 `Path`와 `Files`를 씁니다. 사용법이 직관적이고 코드가 간결합니다.

---

## Path — 경로 다루기

`Path`는 파일이나 디렉토리의 위치를 나타냅니다.

```java
import java.nio.file.*;

// 경로 생성
Path filePath = Path.of("data/hello.txt");           // 상대 경로
Path absPath  = Path.of("/Users/alice/data/hi.txt"); // 절대 경로 (Mac/Linux)
Path winPath  = Path.of("C:\\Users\\alice\\data");   // 절대 경로 (Windows)

// 여러 부분으로 조합
Path combined = Path.of("data", "2024", "log.txt"); // data/2024/log.txt

// 경로 정보 꺼내기
System.out.println(filePath.getFileName());  // hello.txt
System.out.println(filePath.getParent());    // data
System.out.println(filePath.getRoot());      // null (상대 경로라 없음)

// 절대 경로로 변환
Path absolute = filePath.toAbsolutePath();
System.out.println(absolute);  // /현재작업디렉토리/data/hello.txt

// 두 경로 비교
Path p1 = Path.of("data/file.txt");
Path p2 = Path.of("data/file.txt");
System.out.println(p1.equals(p2));  // true
```

## Files — 파일 읽기

`Files` 클래스에는 파일을 읽고 쓰는 static 메서드들이 있습니다.

### 간단한 텍스트 파일 읽기

```java
import java.nio.file.*;
import java.io.IOException;

// 파일 전체 내용을 한 번에 문자열로 읽기 (작은 파일에 적합)
try {
    String content = Files.readString(Path.of("hello.txt"));
    System.out.println(content);
} catch (IOException e) {
    System.out.println("파일 읽기 실패: " + e.getMessage());
}

// 줄 단위로 리스트에 담아 읽기
try {
    List<String> lines = Files.readAllLines(Path.of("data.txt"));
    for (int i = 0; i < lines.size(); i++) {
        System.out.println((i+1) + "번째 줄: " + lines.get(i));
    }
} catch (IOException e) {
    e.printStackTrace();
}
```

### 대용량 파일 — 스트림으로 읽기

파일이 크면 한 번에 읽으면 메모리가 부족할 수 있습니다. 줄 단위로 스트림 처리합니다.

```java
// try-with-resources: 블록 끝에 자동으로 스트림 닫힘
try (Stream<String> lines = Files.lines(Path.of("big.txt"))) {
    lines
        .filter(line -> line.contains("ERROR"))  // 에러 줄만
        .map(line -> line.trim())                 // 공백 제거
        .forEach(System.out::println);            // 출력
} catch (IOException e) {
    e.printStackTrace();
}
```

## Files — 파일 쓰기

```java
// 텍스트 한 번에 쓰기 (파일 없으면 생성, 있으면 덮어씀)
try {
    Files.writeString(Path.of("output.txt"), "Hello, World!\n두 번째 줄");
} catch (IOException e) {
    e.printStackTrace();
}

// 줄 목록 쓰기
List<String> lines = List.of("1번 줄", "2번 줄", "3번 줄");
try {
    Files.write(Path.of("output.txt"), lines);
} catch (IOException e) {
    e.printStackTrace();
}

// 이어 쓰기 (APPEND: 기존 내용 유지하고 덧붙임)
try {
    Files.writeString(
        Path.of("log.txt"),
        "새 로그 항목\n",
        StandardOpenOption.APPEND,    // 이어 쓰기
        StandardOpenOption.CREATE     // 없으면 생성
    );
} catch (IOException e) {
    e.printStackTrace();
}
```

## 파일/디렉토리 관리

```java
Path file = Path.of("data/sample.txt");
Path dir  = Path.of("data/backup");

// 존재 여부 확인
System.out.println(Files.exists(file));        // true / false
System.out.println(Files.isDirectory(dir));    // 디렉토리인지
System.out.println(Files.isRegularFile(file)); // 일반 파일인지

// 디렉토리 생성 (중간 경로도 자동 생성)
Files.createDirectories(dir);  // data 디렉토리도 없으면 함께 생성
Files.createDirectory(Path.of("newDir"));  // 한 단계만, 부모 없으면 에러

// 파일 생성 (내용 없는 빈 파일)
Files.createFile(Path.of("empty.txt"));   // 이미 있으면 에러

// 복사
Files.copy(
    Path.of("original.txt"),
    Path.of("copy.txt"),
    StandardCopyOption.REPLACE_EXISTING  // 대상이 있으면 덮어씀
);

// 이동/이름 변경
Files.move(
    Path.of("old.txt"),
    Path.of("new.txt"),
    StandardCopyOption.REPLACE_EXISTING
);

// 삭제
Files.delete(file);          // 없으면 NoSuchFileException
Files.deleteIfExists(file);  // 없어도 예외 없음

// 파일 크기 (바이트)
long size = Files.size(file);
System.out.println(size + " bytes");

// 수정 시간
FileTime modified = Files.getLastModifiedTime(file);
System.out.println(modified.toInstant());
```

## 디렉토리 탐색

```java
// 바로 아래 자식만 탐색
try (DirectoryStream<Path> stream = Files.newDirectoryStream(Path.of("src"))) {
    for (Path entry : stream) {
        System.out.println(entry);
    }
}

// 재귀 탐색 (하위 디렉토리 포함)
try (Stream<Path> walk = Files.walk(Path.of("src"))) {
    walk
        .filter(Files::isRegularFile)              // 파일만 (디렉토리 제외)
        .filter(p -> p.toString().endsWith(".java"))  // .java 파일만
        .forEach(System.out::println);
}

// 특정 깊이까지만 탐색
try (Stream<Path> walk = Files.walk(Path.of("src"), 2)) {  // 2단계까지
    walk.forEach(System.out::println);
}
```

## 전통 I/O — BufferedReader / BufferedWriter

레거시 코드나 특수한 상황에서 사용합니다. NIO.2보다 코드가 길지만 세밀한 제어가 가능합니다.

```java
// 읽기
try (BufferedReader reader = new BufferedReader(
        new FileReader("hello.txt", StandardCharsets.UTF_8))) {  // UTF-8 인코딩 지정
    String line;
    while ((line = reader.readLine()) != null) {  // null이면 파일 끝
        System.out.println(line);
    }
} catch (IOException e) {
    e.printStackTrace();
}

// 쓰기
try (BufferedWriter writer = new BufferedWriter(
        new FileWriter("output.txt", StandardCharsets.UTF_8))) {
    writer.write("첫 번째 줄");
    writer.newLine();  // 줄바꿈 (OS에 맞게 자동 처리)
    writer.write("두 번째 줄");
} catch (IOException e) {
    e.printStackTrace();
}
```

왜 `BufferedReader`/`BufferedWriter`를 쓸까요? 디스크 접근을 한 번에 많이 해서 속도를 높이기 위한 **버퍼(중간 저장소)** 덕분입니다.

## 자주 하는 실수들

```java
// ❌ 실수 1: try-with-resources 미사용 → 파일 안 닫힘
FileReader reader = new FileReader("file.txt");
String content = reader.readLine();  // 예외 발생 시 close() 안 됨
reader.close();

// ✅ try-with-resources 사용
try (BufferedReader reader = new BufferedReader(new FileReader("file.txt"))) {
    System.out.println(reader.readLine());
}  // 자동으로 닫힘

// ❌ 실수 2: 인코딩 지정 안 함 → 한글 깨짐
Files.writeString(Path.of("korean.txt"), "안녕하세요");  // 기본값 UTF-8 (OK)
new FileWriter("korean.txt");  // 시스템 기본 인코딩 사용 → 환경마다 다름!

// ✅ 인코딩 명시
new FileWriter("korean.txt", StandardCharsets.UTF_8);
Files.writeString(Path.of("korean.txt"), "안녕하세요", StandardCharsets.UTF_8);

// ❌ 실수 3: 파일 존재 여부 확인 없이 삭제
Files.delete(Path.of("maybe-exists.txt"));  // 없으면 NoSuchFileException

// ✅ 안전하게 삭제
Files.deleteIfExists(Path.of("maybe-exists.txt"));

// ❌ 실수 4: 경로 구분자 하드코딩 (OS 호환성 문제)
Path p = Path.of("data\\file.txt");  // Windows에서만 동작

// ✅ Path.of()로 OS 독립적으로
Path p = Path.of("data", "file.txt");  // Windows: data\file.txt, Linux: data/file.txt
```

## 실전 예제 — CSV 파일 읽기/쓰기

```java
record Product(String name, String category, int price) {}

// CSV 파일 읽기
// 파일 내용 예시:
// name,category,price
// 노트북,전자기기,1200000
// 마우스,전자기기,30000

List<Product> readProducts(String filePath) throws IOException {
    return Files.lines(Path.of(filePath))
        .skip(1)  // 헤더 줄 건너뜀
        .filter(line -> !line.isBlank())  // 빈 줄 무시
        .map(line -> {
            String[] parts = line.split(",");
            return new Product(
                parts[0].trim(),
                parts[1].trim(),
                Integer.parseInt(parts[2].trim())
            );
        })
        .collect(Collectors.toList());
}

// CSV 파일 쓰기
void writeProducts(List<Product> products, String filePath) throws IOException {
    List<String> lines = new ArrayList<>();
    lines.add("name,category,price");  // 헤더
    products.stream()
        .map(p -> p.name() + "," + p.category() + "," + p.price())
        .forEach(lines::add);

    Files.write(Path.of(filePath), lines);
    System.out.println(products.size() + "개 상품을 저장했습니다.");
}

// 사용
try {
    List<Product> products = readProducts("products.csv");
    System.out.println("읽은 상품 수: " + products.size());
    products.forEach(p -> System.out.println(p.name() + ": " + p.price() + "원"));
} catch (IOException e) {
    System.out.println("파일 처리 오류: " + e.getMessage());
}
```

## 직렬화 (Java Serialization)

객체를 바이트로 변환해 파일에 저장하거나 네트워크로 전송할 수 있습니다. 단, **보안 취약점**이 있어 실무에서는 JSON(Jackson, Gson)을 선호합니다.

```java
import java.io.*;

// Serializable 마크 인터페이스 구현
class User implements Serializable {
    private static final long serialVersionUID = 1L;  // 버전 관리용
    private String name;
    private int age;
    private transient String password;  // transient: 직렬화에서 제외

    // 생성자, getter...
}

// 객체 저장 (직렬화)
try (ObjectOutputStream oos = new ObjectOutputStream(
        new FileOutputStream("user.dat"))) {
    oos.writeObject(new User("Alice", 30));
    System.out.println("저장 완료");
} catch (IOException e) {
    e.printStackTrace();
}

// 객체 불러오기 (역직렬화)
try (ObjectInputStream ois = new ObjectInputStream(
        new FileInputStream("user.dat"))) {
    User user = (User) ois.readObject();  // 형변환 필요
    System.out.println(user.getName());   // Alice
} catch (IOException | ClassNotFoundException e) {
    e.printStackTrace();
}
```

실무 팁: 파일 저장, API 통신에는 Java 직렬화 대신 JSON을 씁니다.

```java
// Jackson으로 객체 ↔ JSON 변환 (의존성 추가 필요)
ObjectMapper mapper = new ObjectMapper();
String json = mapper.writeValueAsString(user);   // 객체 → JSON 문자열
User loaded = mapper.readValue(json, User.class); // JSON 문자열 → 객체
```
