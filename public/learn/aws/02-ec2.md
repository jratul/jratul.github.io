# EC2 (Elastic Compute Cloud)

가상 서버. AWS 서비스 중 가장 기본.

---

## 인스턴스 타입

```
이름 구조: c6g.xlarge
  c   — 패밀리 (c=compute, m=general, r=memory, t=burstable)
  6   — 세대 (높을수록 최신)
  g   — 프로세서 (g=Graviton ARM, a=AMD, i=Intel, 없으면 Intel)
  .   — 구분자
  xlarge — 크기

주요 패밀리:
t4g, t3 — 버스터블 (저사용률 시 크레딧 축적, 폭발적 사용 가능)
          개발/테스트 서버, 트래픽 불균등한 서비스
m7i, m6i — 범용 (CPU:메모리 = 1:4)
           API 서버, 웹 서버
c7i, c6i — 컴퓨팅 최적화 (CPU:메모리 = 1:2)
           배치 처리, 게임 서버
r7i, r6i — 메모리 최적화 (CPU:메모리 = 1:8)
           DB, Redis, 인메모리 앱
i4i — 스토리지 최적화 (NVMe SSD)
      DB, 대용량 데이터 처리

크기:
nano → micro → small → medium → large → xlarge → 2xlarge → ...
```

## 인스턴스 생성 핵심 요소

```
AMI (Amazon Machine Image):
— OS + 초기 소프트웨어 포함 이미지
— Amazon Linux 2023 (AWS 최적화, 무료)
— Ubuntu 22.04 LTS (가장 많이 씀)
— 직접 만든 커스텀 AMI 사용 가능

키 페어 (Key Pair):
— SSH 접속용 키
— .pem 파일 (개인키), AWS가 공개키 보관
— 분실 시 재발급 불가 → 반드시 안전 보관

보안 그룹 (Security Group):
— 인스턴스 레벨 방화벽
— 인바운드/아웃바운드 규칙
— 화이트리스트 방식 (허용한 것만 통과)
— 동적 변경 가능 (재시작 불필요)

EBS (Elastic Block Store):
— EC2에 연결되는 블록 스토리지
— 기본 8GB (루트 볼륨)
— 인스턴스 종료 시 삭제 여부 설정 가능
```

## 보안 그룹 설정

```
인바운드 규칙 예시 (웹 서버):
유형     프로토콜 포트  소스
SSH      TCP      22   내 IP 또는 Bastion SG
HTTP     TCP      80   0.0.0.0/0 (전체)
HTTPS    TCP      443  0.0.0.0/0
Custom   TCP      8080 ALB 보안 그룹

인바운드 규칙 예시 (RDS):
PostgreSQL TCP    5432 앱 서버 보안 그룹

원칙:
— SSH는 특정 IP만 허용 (0.0.0.0/0 절대 금지)
— DB는 앱 서버 보안 그룹에서만 허용
— 불필요한 포트 개방 금지
```

## EC2 접속

```bash
# SSH 접속
chmod 400 my-key.pem
ssh -i my-key.pem ubuntu@ec2-xxx.ap-northeast-2.compute.amazonaws.com
ssh -i my-key.pem ubuntu@1.2.3.4    # 공인 IP로 접속

# SSH Config 설정 (편의)
# ~/.ssh/config
Host myserver
    HostName 1.2.3.4
    User ubuntu
    IdentityFile ~/.ssh/my-key.pem

ssh myserver    # 이후 짧게 사용
```

## EC2 기본 운영

```bash
# 인스턴스 시작/중지/종료
aws ec2 start-instances --instance-ids i-1234567890abcdef0
aws ec2 stop-instances --instance-ids i-1234567890abcdef0
aws ec2 terminate-instances --instance-ids i-1234567890abcdef0

# 인스턴스 정보
aws ec2 describe-instances --instance-ids i-1234567890abcdef0

# 인스턴스 내부에서 메타데이터 조회
curl http://169.254.169.254/latest/meta-data/instance-id
curl http://169.254.169.254/latest/meta-data/instance-type
curl http://169.254.169.254/latest/meta-data/public-ipv4
```

## User Data — 시작 시 자동 실행 스크립트

```bash
#!/bin/bash
# EC2 시작 시 자동 실행 (한 번만)

# 패키지 업데이트
apt update -y

# Java 설치
apt install -y openjdk-17-jdk

# 앱 디렉토리 생성
mkdir -p /opt/myapp /var/log/myapp

# 앱 다운로드 (S3에서)
aws s3 cp s3://my-deploy-bucket/app.jar /opt/myapp/app.jar

# 서비스 등록
cat > /etc/systemd/system/myapp.service << EOF
[Unit]
Description=My Spring Boot App
After=network.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/opt/myapp
ExecStart=/usr/bin/java -Xmx512m -jar /opt/myapp/app.jar
Restart=always
Environment=SPRING_PROFILES_ACTIVE=prod

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable myapp
systemctl start myapp
```

## Elastic IP

```bash
# 고정 공인 IP 할당
aws ec2 allocate-address --domain vpc
# → AllocationId: eipalloc-xxx

# 인스턴스에 연결
aws ec2 associate-address \
  --instance-id i-1234567890 \
  --allocation-id eipalloc-xxx

# 주의: Elastic IP를 인스턴스에 연결하지 않으면 요금 발생!
# 사용하지 않는 Elastic IP는 즉시 반환
aws ec2 release-address --allocation-id eipalloc-xxx
```

## EC2 Auto Scaling

```
Auto Scaling Group (ASG):
— 트래픽에 따라 인스턴스 수 자동 조절
— 최소/최대/원하는 인스턴스 수 설정
— Launch Template으로 인스턴스 구성 정의

스케일링 정책:
Target Tracking — CPU 70% 유지 (권장)
Step Scaling    — 임계값에 따른 단계별 증감
Scheduled       — 예측 가능한 트래픽 패턴

구성:
ALB → Target Group → Auto Scaling Group
                         ├── EC2 (AZ-a)
                         ├── EC2 (AZ-b)
                         └── EC2 (AZ-c)
```

```bash
# Launch Template 생성
aws ec2 create-launch-template \
  --launch-template-name my-app-lt \
  --version-description "v1" \
  --launch-template-data '{
    "ImageId": "ami-xxx",
    "InstanceType": "t3.small",
    "KeyName": "my-key",
    "SecurityGroupIds": ["sg-xxx"],
    "UserData": "base64-encoded-script",
    "IamInstanceProfile": {"Name": "my-instance-profile"}
  }'

# Auto Scaling Group 생성
aws autoscaling create-auto-scaling-group \
  --auto-scaling-group-name my-app-asg \
  --launch-template LaunchTemplateName=my-app-lt,Version='$Latest' \
  --min-size 2 \
  --max-size 10 \
  --desired-capacity 2 \
  --vpc-zone-identifier "subnet-xxx,subnet-yyy" \
  --target-group-arns arn:aws:elasticloadbalancing:...
```

## 인스턴스 구매 전략

```
온디맨드:
— 트래픽이 불규칙한 워크로드
— 새로운 서비스 (수요 예측 어려울 때)

Reserved (약정):
— 베이스 트래픽 처리 (항상 필요한 용량)
— 1년: 40% 할인, 3년: 60% 할인

Spot:
— 추가 트래픽 처리 (베이스 + Spot 혼합 ASG)
— 배치 작업, 데이터 처리
— 인터럽트 감내 설계 필요

실전 조합:
- 베이스 2개: Reserved (저렴)
- 스케일 아웃: Spot (최대 90% 절감)
- Spot 중단 시: 온디맨드로 fallback
```
