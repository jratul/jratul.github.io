# S3 (Simple Storage Service)

무한 확장 가능한 객체 스토리지. 파일 저장, 정적 웹 호스팅, 데이터 레이크에 활용.

---

## 기본 개념

```
버킷 (Bucket):
— 객체를 담는 컨테이너
— 이름은 전 세계에서 유일해야 함
— 리전에 종속

객체 (Object):
— 실제 파일 + 메타데이터
— 최대 5TB
— Key = 파일 경로 (예: images/2024/photo.jpg)

특징:
— 99.999999999% (11 9's) 내구성
— 버전 관리 지원
— 수명 주기 정책 (자동 삭제/전환)
— 이벤트 알림 (Lambda, SQS, SNS)
```

---

## 기본 사용

```bash
# 버킷 생성
aws s3 mb s3://my-unique-bucket-name --region ap-northeast-2

# 파일 업로드
aws s3 cp file.txt s3://my-bucket/
aws s3 cp file.txt s3://my-bucket/subdir/
aws s3 sync ./local-dir s3://my-bucket/remote-dir/

# 파일 다운로드
aws s3 cp s3://my-bucket/file.txt ./
aws s3 sync s3://my-bucket/remote-dir/ ./local-dir/

# 목록 확인
aws s3 ls s3://my-bucket/
aws s3 ls s3://my-bucket/ --recursive

# 삭제
aws s3 rm s3://my-bucket/file.txt
aws s3 rm s3://my-bucket/ --recursive
```

## 스토리지 클래스

```
Standard:
— 자주 접근하는 데이터 (기본)
— 가용성 99.99%
— 비용: 저장 $0.023/GB, 조회 $0.0004/1000req

Standard-IA (Infrequent Access):
— 월 1회 이하 접근
— 저장 저렴, 조회 시 추가 비용
— 30일 이상 저장 시 이점

One Zone-IA:
— 단일 AZ (가용성 낮음)
— IA보다 20% 저렴
— 재생성 가능한 데이터에만 사용

Glacier Instant Retrieval:
— 아카이브 (분기 1회 접근)
— 밀리초 조회
— IA보다 저렴

Glacier Flexible Retrieval:
— 아카이브 (연 1회 접근)
— 조회 시간: 1분 ~ 12시간
— Instant보다 저렴

Glacier Deep Archive:
— 장기 보관 (7년+)
— 가장 저렴 ($0.00099/GB)
— 조회 12 ~ 48시간

Intelligent-Tiering:
— 접근 패턴에 따라 자동 전환
— 접근 패턴 모를 때 사용
```

## 수명 주기 정책

```json
{
  "Rules": [
    {
      "Id": "archive-old-files",
      "Status": "Enabled",
      "Filter": {"Prefix": "logs/"},
      "Transitions": [
        {
          "Days": 30,
          "StorageClass": "STANDARD_IA"
        },
        {
          "Days": 90,
          "StorageClass": "GLACIER"
        }
      ],
      "Expiration": {
        "Days": 365
      }
    }
  ]
}
```

```bash
# CLI로 수명 주기 설정
aws s3api put-bucket-lifecycle-configuration \
  --bucket my-bucket \
  --lifecycle-configuration file://lifecycle.json
```

## 버킷 정책 (Bucket Policy)

```json
// 퍼블릭 읽기 허용 (정적 웹사이트)
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicRead",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::my-bucket/*"
    }
  ]
}

// 특정 IAM Role만 허용
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "AWS": "arn:aws:iam::123456789:role/my-app-role"
      },
      "Action": ["s3:GetObject", "s3:PutObject"],
      "Resource": "arn:aws:s3:::my-bucket/*"
    }
  ]
}
```

## Presigned URL

```java
// Spring Boot에서 Presigned URL 생성

@Service
@RequiredArgsConstructor
public class S3Service {

    private final S3Presigner presigner;
    private final S3Client s3Client;

    @Value("${aws.s3.bucket}")
    private String bucket;

    // 업로드용 Presigned URL (클라이언트가 직접 S3에 업로드)
    public String generateUploadUrl(String key, Duration expiry) {
        PutObjectPresignRequest presignRequest = PutObjectPresignRequest.builder()
            .signatureDuration(expiry)
            .putObjectRequest(r -> r.bucket(bucket).key(key))
            .build();

        return presigner.presignPutObject(presignRequest)
            .url().toString();
    }

    // 다운로드용 Presigned URL
    public String generateDownloadUrl(String key, Duration expiry) {
        GetObjectPresignRequest presignRequest = GetObjectPresignRequest.builder()
            .signatureDuration(expiry)
            .getObjectRequest(r -> r.bucket(bucket).key(key))
            .build();

        return presigner.presignGetObject(presignRequest)
            .url().toString();
    }

    // 서버에서 직접 업로드
    public void upload(String key, byte[] data, String contentType) {
        s3Client.putObject(
            PutObjectRequest.builder()
                .bucket(bucket)
                .key(key)
                .contentType(contentType)
                .build(),
            RequestBody.fromBytes(data)
        );
    }

    // 삭제
    public void delete(String key) {
        s3Client.deleteObject(b -> b.bucket(bucket).key(key));
    }
}
```

```java
// 설정
@Configuration
public class S3Config {

    @Bean
    public S3Client s3Client() {
        return S3Client.builder()
            .region(Region.AP_NORTHEAST_2)
            .credentialsProvider(DefaultCredentialsProvider.create())
            .build();
    }

    @Bean
    public S3Presigner s3Presigner() {
        return S3Presigner.builder()
            .region(Region.AP_NORTHEAST_2)
            .credentialsProvider(DefaultCredentialsProvider.create())
            .build();
    }
}
```

## 정적 웹사이트 호스팅

```bash
# 정적 호스팅 활성화
aws s3api put-bucket-website \
  --bucket my-bucket \
  --website-configuration '{
    "IndexDocument": {"Suffix": "index.html"},
    "ErrorDocument": {"Key": "error.html"}
  }'

# 퍼블릭 액세스 허용
aws s3api put-public-access-block \
  --bucket my-bucket \
  --public-access-block-configuration \
    BlockPublicAcls=false,IgnorePublicAcls=false,\
    BlockPublicPolicy=false,RestrictPublicBuckets=false

# 버킷 정책 (퍼블릭 읽기)
aws s3api put-bucket-policy \
  --bucket my-bucket \
  --policy file://public-policy.json

# 접속 URL: http://my-bucket.s3-website.ap-northeast-2.amazonaws.com
# CloudFront + ACM 인증서로 HTTPS 적용 권장
```

```bash
# S3 이벤트 알림 설정
aws s3api put-bucket-notification-configuration \
  --bucket my-bucket \
  --notification-configuration '{
    "LambdaFunctionConfigurations": [
      {
        "LambdaFunctionArn": "arn:aws:lambda:...:function:my-function",
        "Events": ["s3:ObjectCreated:*"],
        "Filter": {
          "Key": {
            "FilterRules": [
              {"Name": "prefix", "Value": "uploads/"},
              {"Name": "suffix", "Value": ".jpg"}
            ]
          }
        }
      }
    ]
  }'
```
